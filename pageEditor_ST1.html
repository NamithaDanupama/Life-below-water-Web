Splash Page

Technical Description

  • <img> to use a background image.
  • Full-screen background using height: 100vh; width: 100%;.
  • Fade in and fade out animations using @keyframes.
  • <div> and class uses to make boxes CSS and hTMl.

Accessibility

  • Color Contrast & Readability: Ensured high color contrast between text and background to improve readability for users with visual impairments.
  • Screen Reader Compatibility: Added appropriate ARIA labels and alternative text for images and buttons to support screen readers.
  • Keyboard Navigation: Made sure users can navigate through the splash screen using only a keyboard (Tab, Enter, etc.).

Volunteer Page

Technical Description

  • Card layout using <div class="volunteer-card">.
  • Hover effects for buttons using: :hover{}.
  • Styling buttons and boxes using border-radius.
  • Use links to make ui beautiful using <a href=""> tag.
  • Use imojis to make a rating <header>
  • Use a global HTML and CSS file for the <header>

Accessibility

  • Form Accessibility: Used proper labels and placeholder text for input fields, with ARIA labels like aria-label="Full Name" and aria-label="Email Address" to enhance screen reader support.
  • Responsive Design: Ensured the page adapts to different screen sizes, making it accessible on mobile and larger displays.
  • Descriptive Links & Buttons: Used meaningful text for buttons and links (e.g., “Apply Now” instead of “Click Here”) to aid users relying on assistive technologies.

Content Page

Technical Description

  • Make a dropdown list to go through sections <class="dropdown">.
  • Use id="" to make section and use to link them.
  • Making a "go to top button" with hover::after to make a text appear on the button when hovers.
  • Use the basic <footer> area to make the page more user friendly.

Accessibility

  • Text Resizing & Readability: Enabled relative font sizing so users can adjust text size as needed for better readability.
  • Alt Text for Images: Added descriptive alt text to images to ensure visually impaired users understand the content.
  • Semantic HTML Structure: Used proper heading levels (H1, H2, H3) and landmarks to improve navigation for screen reader users.

Challenges and Lessons Learned

  • Challenges :
    • Layout issues - The elements misplaced itself by missusing the CSS eliments.
    • CSS styling conflicts - Some styles just didn't work as planned
    • Font and icon integration - some external fonts and icons failed to load correctly.
  • Solutions :
    • Used Flexbox and Grid to use the proper layout.
    • Ensured CSS specificity and use id and classes .
    • Ensured correct links for fonts and icons.
  • Lessons :
    • Flexbox and Grid makes the work easy and clean.
    • Proper CSS use keep the code clean and proper working.
    • Always double-check external resource links.

Compliance

  • Accessibility: Our pages meet Web Content Accessibility Guidelines (WCAG), ensuring all users, including those with disabilities, can access content. This includes text alternatives for images, proper color contrast, and easy navigation via keyboard.
  • Data Protection: We comply with GDPR regulations, ensuring user data is collected with clear consent and securely handled. All sensitive information is transmitted with encryption via SSL.
  • Security: We prioritize security by using secure authentication methods and safeguarding against vulnerabilities like SQL injection and XSS, ensuring a safe browsing experience.
